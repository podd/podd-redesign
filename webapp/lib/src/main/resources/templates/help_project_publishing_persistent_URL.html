Persistent URLs


This section describes a policy and system process for assigning PURLs to objects that are to be published. It does not include any screen based processes.

Persistent URLS (PURLs) is used to maintain a unique identity for any object managed by PODD.

ANDS has a subscription to the global Handles infrastructure. For example, an ANDS PURLs http://hdl.handle.net/102.100.100/abc has a static domain name http://hdl.handle.net/ and a unique handle to the object (i.e. 102.100.100/abc).

The generation and registration of PURLs are managed through tools provided by ANDS.

PODD includes PURL fields as part of the core metadata elements for any object and will require a web service for object retrieval through a PURL.

* Business Rule: One object equals one Persistent URL. Even though PURLs can be treated as an array, the index elements cannot be resolved in a URL.

Search for Users


In the Project Participant section, the headers for each of the input fields actually links to a search dialog.

When the user clicks on any of the project participant input field headings (i.e. Principal Investigator, Project Administrators, etc) on the Edit Project screen, a pop up dialog will appear.

The flow for using the dialog is as follows:

  1. The list of PODD users will list all available PODD users as a default.
  2. The list will filter as the user starts to type in the field.
  3. The list can be sorted using the Sort By drop down list. The default is First Name, other options are Last Name, Institution, Email Address.
  4. Check boxes are available next to each entry in the list of users. When the user checks these boxes the selection list will be updated to include the selected participant. If the boxes are unchecked then the participant would be removed.
  5. Note that the title of the selection list will depend on the link used to access the dialog. Therefore it could be either Principal Investigator, Project Administrators, Project Members or Project Observers.
  6. The rule of one Principal Investigator must be adhered to. Only one PI can be selected. The mechanism should be that if a second PI is checked then the first PI is automatically unchecked. An alternative mechanism would be to display an error message.
  7. When the user is satisfied they click Submit.
  8. The relevant text field in the Project page will be added to (Note: not replaced!).
  9. The user may click Cancel if they wish to abandon the dialog and not update the main page.

Submit, Reset, Cancel or Reinitialize


At the bottom of the Create Project Page, there are four buttons displayed (Figure 10.1)

  • Submit
  • Reset
  • Cancel
  • Reinitialize

The user can select one of the buttons based on the job requirement .


Submit

The user selects Submit. All the information entered in the previous three sections is submitted to the system. Validation is performed.

If any fields are in error then the user is returned to the top of the page. An error box will appear at top of the page advising there are errors on the page. An error box will also appear beneath the fields in error describing the nature of the error and remedial action.

If no error is detected, the system confirms the submission to the user by switching to the view tab and displaying the project details page in View Project mode (See: Viewing a Project for details).

An email is sent to the Principal Investigator and Project Administrator, advising them that a project has been created for them and that they may now update the project. No response is required from the user. The following additional values will be automatically recorded against the Project concrete object:

  • Project ID, which is determined by PODD,
  • Creator is set to the current user,
  • Creation Date is set to the current date,
  • Project Status is set to 'Current',
  • Publication Status is set to Not Published,
  • Last modified by is set to the current user,
  • Last modification date is set to the current date.

Please note that where a project is created by a Project Creator and they have removed themselves from the Project Participant fields, they will not be able to view the project details later. In this circumstance they are not directed to the View Project page. A screen advising them they have insufficient access to the project.


Reset

The user can also reset the Create Project page by selecting the Reset button. It means that all details entered will be cleared. The screen remains on the Create Project page. The current user is listed in the Principal Investigator field. The user can then re-enter the project information and submit the project when satisfied.


Cancel

If the user clicks the Cancel button, the Create Project page is abandoned. All information entered will be lost, including the uploaded data files. New Project is not created and the user is then returned to the List Project Page.


Reinitialize

This button works when another Project/Object is being created or edited by current user. As each user can only edit one object each time, the user need to reinitialize the system to ensure the current object can be created.

If the user fills in all mandatory fields and clicks Submit when another object is being created/edited, the user will be redirected to a new screen advising that another object is being edited. The Reinitialize button will be available on the screen (see Figure 10.4.1).

Figure 10.4.1 – An example screen shot advising the user that another object is being created/edit and reinilialization is required.

Once the ReInitilaze button is clicked, the user is returned to the Create Project page. The Project can then be submitted.

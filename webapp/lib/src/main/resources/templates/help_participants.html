Adding Project Participants


Project Participants are users who have Project specific roles. They have nominated CRUD accesses to the project while the project has a Project Status of current.

Figure 10.2.1 – screen shot of the Project Participants section on the Create Project Object page

As shown in Figure 10.2.1, there are four types of project participants (see also Project Specific Roles). They are:

  • Principal Investigator *
    • Mandatory, a project must have a Primary Investigator.
    • A Primary Investigator, by default, is also a Project Administrator.
    • No more than one Primary Investigator at any time.
    • Primary Investigators can be replaced.
    • Additional text under the text field should read: "Only one principal investigator is allowed. Principal Investigators will have Project Administrator status by default."
  • Project Administrator
    • The project can have multiple Project Administrators.
    • A default Project Administrator is already inserted in this field, being the person creating the project. This default can be edited out of the text field. Note this will have consequences when the creator submits the project.
    • Additional text under the text field should read: "Project Administrators will have Create, Read, Update and Delete access to all Project objects."
  • Project Members
    • Additional text under the text field should read: "Project Members will have Create, Read and Update access to all Project objects."
  • Project Observers
    • Additional text under the text field should read: "Project Observers will have Read only access to all Project objects."

The project participants section also provide the following two functionalities:

  • Auto-Completion: the headers for each of the input fields links to a search dialog. They provide a selectable auto-completion functionality.
  • Search for User: When the user clicks on any of the input field headings (i.e. Principal Investigator, Project Administrators, etc) a pop up dialog will appear.

The user adds user to each field as required. Please be advised that only existing registered user can be added in the fields. One user can have only one role in a project.

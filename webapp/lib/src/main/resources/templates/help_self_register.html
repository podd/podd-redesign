Self Registration

A public user can create their own user account on the system.

When navigating to the PODD website, the following screen is displayed (see Figure 4.1.1).

Figure 4.1.1 – Screen shot of PODD home page

The registrant clicks either of the Login links on the screen, the following screen is displayed (Figure 4.1.2).

Figure 4.1.2 – Screen for the user to log on or self register

The LHS is the AAF login window, and the RHS is the PODD. In the RHS a Register button is displayed on the page for the public user to create an account for themselves. Once the button is hit, a User Self Registration screen is returned (Figure 4.1.3).

 Figure 4.1.3 Sample screen of User Self Registration

As shown on Figure 4.1.3, the following procedure is displayed:

  • The registrant finishes the account information first, including User name, Email address, Password and Confirm password;
  • The status radio buttons are not selectable (default to active) and the Roles radio buttons are not selectable either. The role will default to Repository User and can only be changed by a Repository Administrator;
  • The registrant fills in personal information: Title (optional), First name, Last name, Organisation/Institution, phone number, mailing address and url (optional);
  • The registrant completes the CAPTCHA test;
  • The the registrant clicks the Create New User button.

Validation is performed. An error message will be displayed if any field is in error. Otherwise, the user will receive an email to say they have registered successfully.

If the default status is 'Active' they will be redirected to the login page and a message will be displayed telling them their self registration was successful and they can now log in. (But if the default status is 'Blocked' they will receive a message advising that their login account is currently pending approval by the site support).

Self registered users are created as repository users. By default, a self registered user has no greater access rights than an anonymous user, with the exception that they can provide annotated comments against publicly available projects.

Note that the repository administrator can disable the self-registration function. That way, the Register button shown in Figure 4.1.2 won't be displayed. The repository administrator can also change the default status of a self registered user between blocked and active.

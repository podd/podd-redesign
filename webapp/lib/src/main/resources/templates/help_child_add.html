Adding a Child Object


With the exception of Project, all concrete objects are instantiated with a reference to the parent object. But not all objects can be a parent of another object, e.g., a Project Plan can not have a child object.

Authorised users can add a child object by clicking the Add Child Object button when viewing the parent object.

If the data model does not allow an object to have child objects, the Add Child Object button won't be available.


Select Parent-Child Relationship and Child Object Type

The user chooses to create a child object as described above, the initial Add Child Object screen is displayed as shown in Figure 14.1.1.

Figure 14.1.1 – Example screen shot for adding a child object & relationships to a parent object (Project)

The first section on the screen is a summary of the Parent Object including (read only):

  • ID (Note: ID must be displayed).
  • Type (i.e. object type)
  • Title

* The user can get to the parent object view page by clicking the Object ID link.

The next section is the Relationship & Type section. The Relationship defines the nature of the relationship (predicate) between the parent (subject) and the child (object) object. The user selects the relationship  and the type of object.

  • These fields are drop-down lists at all times. If there is only one option then it will still be a drop down, but the single option will be selected.
  • The Relationship field will display the Title of the predicate and not the predicate itself.
  • The Child Object list will be dynamically generated when the user selects the Relationship field.

The next section is the Continue and Cancel buttons.

When satisfied with the relationship and child object the user clicks Continue. (If Cancel is clicked, the Add Child Object action is then cancelled and the user is returned to the previous View Object page).

The user is presented with an updated Add Child Object screen. The screen is different depending on the type of the child object.

 

Figure – 14.1.2 Example screen shot for updated add a child object page (Platform Object)


Fill in the Child Object details

As shown in Figure 14.1.2, the Add Child Object page has the following sections:

  • Parent Details
  • Object Details
  • Refers To tables (can be zero or more than one table depending on the child object type)
  • File Attachment
  • Action buttons

Parent Details: this section is displayed on the top of the page. It is similar to the screen described in the previous screen, including the parent object’s ID, type and title. It also includes the Parent Child object relationship, which has been set to Read Only fields based on the choices the user made in the previous screen. 

Object Details: The next section on the page is the Child Object Details section. Its sub-header is based on the object that is being instantiated, so a Platform object is titled Platform Details. The fields here are determined by the object to be instantiated. Different object has deferent object details. The user completes the fields as required.

Refers To table: Whether this section is displayed or not is determined by the data model and the availability of the objects related. If the object can refer to any objects, the tables and the available referencing objects will be listed. The user then selects the right referenced object.

File Attachment: this functionality is exactly the same as the one when Creating a Project.

Action Buttons: the Submit, Cancel, Reset and Reinitilize buttons are displayed at the bottom of the page. When satisfied with the object details the user clicks Submit. Validation is performed. If any fields are in error then the user is returned to the top of the page. An error box will appear at top of the page advising there are errors on the page. An error box will appear beneath the fields in error describing the nature of the error.

On successful instantiation, the following universal attributes will be automatically recorded against the concrete object:

  • ID is determined by PODD.
  • Creator is set to the current user.
  • Creation Date is set to the current date.

The system confirms the submission to the administrator by switching to the View tab and displaying the object details page in view (read only mode). See: Viewing an Object for details.

The user can also backout and return to the previous screen by selecting the Cancel button or clear the object details by clicking the Reset button. If there is another object being edited at the same time, the user has to click Reinitialize to add the Child Object.

Editing a Project


Overview

The Edit Project screen is displayed as either a consequence of clicking the Edit button from the Project view page  or as a result of selecting the Edit link next to the Project object in the Browser Pane.

The user is presented with the Edit Project screen.  The following sections can be edited by authorised users on this page

  • Project Details
  • Project Participants
  • Attach Files

Three action buttons, Submit, Cancel and Reset, are listed at the bottom of the page.


Editing the Project Details

As shown in Figure 12.1, the following fields are available in the Project Details section for editing:

  • Title: *
  • Description: *
  • Lead Institution: *
  • Start Date: *
  • Duration: (expected duration, free form text)
  • Project Status (not available for project member): * Drop down list, select from Current/Inactive/Complete.

Figure 12.1 – Screen shot of Editing Project details on Edit Object screen

Creator, Creation Date, Last modified by and Last modified date are displayed in a Read Only mode.

The Repository administrator and the Project Administrator /Principle Investigator can change all project details. A project member can modify the Project Title, description, lead institution, Start date and duration, but NOT the Project Status.


Updating the Project Participants

Only Repository Administrator and Project Administrator / Principle Investigator can update the Project Participants. Figure 12.2 displayed the Project Participants section on the Edit Object screen. This section is not visible to the project member when the project member edits a Project.

For the project participant fields, users can modify the existing participants as described previous sections: Creating a Project.

Figure 12.2 – Updated Project Participants fields in Edit Project Screen

The following rules apply to this section:

  • Principal Investigator must have only one valid participant. It can be replaced.
  • If the user is not the project administrator, the user will be added to the Project Administrator field automatically.
  • The text fields only accept valid participants and participants are separated by line returns.
  • A participant can only be associated with a project once. If a username is applied in two fields then an error should occur.
  • Existing participants can be removed from the fields, new New users can be added to the field.

Attaching Files

This section is the same as per Attaching Files in Creating a Project. More Files can be attached. Existing Files are listed (Figure 12.3).

Figure 12.3 – Screen shot of the Attach Files section on the Edit Object screen

Repository Administrator, Project Administrator / Principle Investigator and Project Member all have the same access to this section.


Validation

The user clicks Submit.  Validation is performed.  If any fields are in error then the user is returned to the top of the page. An error box will appear at top of the page advising there are errors on the page. An error box will also appear beneath the fields in error describing the nature of the error and remedial action.
The system confirms the submission to the administrator by switching to the view tab and displaying the project details page in view (read only mode).
The following additional values will be automatically updated against the Project concrete object:

  • Last Modified by is set as the current user
  • Last Modified on is set to the current date
  • Completion Date is set to the current date if the Project Status is set to complete.

The user may click Cancel instead of Submit to abandon the Edit Screen and get back to the View Screen.

The user may click Reset to change the modified information back to the original values and stays on the Edit Screen.

Auto-completion Functionality


User Auto-completion is a very useful function in PODD. The system lists all users which matches the characters typed in the field. The user can then select the Project Participant from the list without typing all the complete user name, institution and email address. 

Figure 10.2.2.1 shows an example of the functionality. When "f" is typed in the Project Member field, a drop down list which displays all users with letter "f" in their user name, institution and email address pops up. So the user can select the disired user to the field.

 

Figure 10.2.1.1 – Detail showing selectable list to autocomplete fields

The detailed process of the auto-completion functionality is described as follows:

  1. The user starts to type in the required field.
  2. For the first letter, number or symbol typed the system will list all available users that:
  3. Has a first name or first name component that starts with the letter (a name component is any part of a name separated by a space or a symbol, e.g. Yuan-Fang has two name components, Yuan and Fang);
  4. Has a last name or last name component that starts with the letter;
  5. Has an institution or institution name component that starts with the letter;
  6. Has an email address or email address component that starts with the letter, number or symbol (email address components are separated by @ and .).
  7. As the user types in subsequent letters, numbers or symbols the list will be further filtered;
  8. The list will be in the form of a drop down box below the required text field;
  9. The first name on the list will be highlighted;
  10. The letters, numbers, etc, in the first part of the user details that match what is being typed in will be bolded in the list.
  11. The user can use the up and down arrow keys to select the desired project participant;
  12. The user hits enter once the selection is made (or clicks on a selection), the field where the letters, etc, were being typed is now updated with the full name, institution and email address of the participant, followed by a line return, which will delimit this participant and the next participant;
  13. The input fields are static sizes, if the Project Administrators, Project Members or Project Observers fields fill up then a scroll bar should appear (standard text field behaviour).
  14. Validation is not performed until the user clicks Submit at the bottom of the page. Fields must be validated as there is no mechanism to prevent the user from mangling a field after it is auto-completed.

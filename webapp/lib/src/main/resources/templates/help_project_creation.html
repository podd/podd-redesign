Creating a Project


Overview

PODD organizes all data into projects. To record any data or metadata, the first thing is to create a concept project object.


Actors

  • Public users and registered users can’t create projects.
  • Project Creators may create projects. They will not have any further access over the project unless they define themselves with a project role during project creation. Note the default behaviour of the system is to define the Project Creator with a Project Administration role. This can be edited out.
  • Administrators may create and modify projects.

Creating a Project

For the authorised users, i.e., the Project Creator and Repository Administrator, there is a Create Project button listed on the Listing Projects screen. Once the button is hit, the Create Project page is displayed. 

 

Figure 10.1 – Screen shot of the Create Project Object page in a collapse state

Figures 10.1 shows an example screen of the Create Project Object Page. It is in an expand state by default. Here it is displayed in a collapse state to save space. There are three sections on the Create Project page:

  • the project details section
  • the project participants section
  • the file attachment section

A collapse/expand icon is listed next to the heading of each section. By clicking this icon, the section is switched bwtween collapsing and expanding state. All the three sections are displayed in expanding status by default when Create Project Object screen is initated. They are displayed in a collapsing statue here to reduce the size of the figure. Each section will be shown in an expanding state and discribed in details separately in the sub-sections listed at the bottom of this page.

At the bottom of the page, four buttons are listed for user to take some actions. They are: Submit, Reset, Cancel and Reinitialize.

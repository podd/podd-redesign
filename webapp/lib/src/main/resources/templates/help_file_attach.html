Attaching Files


Overview

The third section on the Create Project Object page is the File Attachment section. A user may choose to add some files as data items to the project through this section.

A Data Item is considered to be a data stream (i.e. a file) attached to a concrete PODD object. One concrete PODD object can have zero to many Data Items. Each data item carries its own metadata.

Note that Data streams are attached to the relevant PODD object (e.g. Measurement, Analysis, etc), not the PODD data object.


Uploading Data Files

Figure 10.3.1 – Example screen shot for an Attach File dialog

Figure 10.3.1 shows a clean screen of an Attach Files dialog. Before attaching the files, the user may enter some description to accompany the files in the File Description field.

To attach a file, the user clicks the Browse button to open a standard browser File Upload dialog window. A single file or multiple files can be opened in the File Upload dialog. The file location is then filled with the file name, size, directory (location of the file on the user's system or network) and modified date (see Figure 10.3.2).

Figure 10.3.2 – Example screen shot for an Attach File dialog with two files selected for uploading

At this stage the user may refine the list by removing all the files, or highlighting some of the files and clicking the Remove Selected button.

Once satisfied with the files selected, the user then clicks the Upload button and the files are attached and the description is attached to the files uploaded. Note that the user can also click the Stop button to cancel the upload before the Upload process is completed.

In the load process a data stream is created for the current PODD object. Default metadata fields for the data item, such as Date, Type, Format, File Size, are determined from the file (see viewing attached files).

* Note that the data items uploaded are not saved unless the Submit button at the bottom is clicked.


Managing the Uploaded files before Submitting the Object

 

Figure 10.3.3 – Example screen of the Attach File section (with some files attached already and some files opened for upload)

Uploaded files are listed under the Attach new File dialog (see Figure 10.3.3). More files can be uploaded as attachment if necessary. For the files uploaded already:

  • By default they are listed in order of uploading.
  • A red cross icon is displayed next to each uploaded file. By clicking the red-cross icon, individual file can be removed from the section.
  • The files uploaded can also be replaced. If the user uploads a new file with the same name as a previously uploaded one, the new one would replace the existing file with a new version. The original file is retained by Fedora-Commons with a version number if submitted, but there is no provision for users to access earlier versions of the file.

After submitting, the uploaded files/ data items are then saved in PODD as attachement to the PODD object. They can be downloaded, viewed, edited or deleted by authorised users later (see Managing Data Items).

Filtering Projects


Overview

Filtering Projects is a persistent function that can be applied to a project list so that a registered user can determine which projects they see in their project listing.


Actors

  • Public users can not filter projects
  • Registered users can filter Public Projects and My Projects
  • Administrator can filter All Projects

For registered users the filter will persist across sessions.


Filtering the Project List

To filter the project list the registered user first selects the Filter Projects button on the Project List View. The button is not visible to public users.

The view in the Projects tab will change to the following Filter Projects Dialog (see Figure 9.1).

Figure 9.1 – Filter Projects Dialog

The user may enter any combination of the following information:

  • Project Type: Check boxes for My Projects and Public Projects (checked by default).
  • Filter Terms (these are search terms applied to project metadata fields only, e.g. Title, Description, Lead institution, etc).
    • Containing all these words: Is equivalent to joining search terms with a logical and operator. So a search on salt water will be treated as salt and water.
    • Containing any of these words: Is equivalent to joining search terms with a logical or operator. So a search on salt water will be treated as salt or water.
    • Containing the phrase: Is equivalent to wrapping the search terms in quotes and searching for the exact phrase.
    • Excluding these words: Will exclude results containing these words.
  • Filter Dates: There are two sets of date fields. The user may filter for dates falling between the from and to fields, or the user may search only after a particular date by using the From field only, or only before a particular date by using the To field only. The user can select which relevant project date field to search from, the options are:
    • Start Date
    • Completion Date
    • Publication Date

When Reset button is clicked, the system will reset all fields to the default values.

When the Filter Button at the bottom of the page is clciked, the Project Listing view in the Projects Pane with the filtered list of projects will be returned.

Figure 9.2 – Filtered Project List View (Project Creator)

Projects so filtered are still separated between My Projects and Public Projects. On this page, a new button, Remove Filter, is displayed next to the Filter Projects button.


Persistence of Filters

For a registered user a filter persists during and between sessions. If a registered user sets a project filter, then anytime the user visits the Projects View, the view will be filtered as per the filter the user last created.

When a user has a persistent filter in place, if they go to List Projects view the project list will be filtered as the user had filtered them the last time they submitted the Filter Projects dialog.

When a user has a persistent filter in place, if they go to the Filter Projects dialog, the fields will be populated as the user had populated them the last time the user submitted this dialog.

When a user has a persistent filter in place, the user will see a Remove Filter button on the List Projects view. If there is no persistent filter in place, the user will not see this button.


Resetting the Filter

The user can reset the filter in two ways:

  • By clicking the Remove Filter button on the List Projects view. This will change the List Projects view to display all projects the user may access: My Projects and Public Projects.
  • By clicking the Filter Projects  button in the Projects tab and then selecting the Reset button in the Filter Project dialog. This will reset all fields to their default values. The user then clicks the Filter button, the system will list all projects the user may access, including My Projects and Public Projects.
  • If a repository administrator wants to filter all project he can access, then both My Projects and Public Projects should be unchecked.

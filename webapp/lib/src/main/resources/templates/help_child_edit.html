Editing a Child Object


The Edit Object screen is arrived at by selecting the object to edit from the Browser Pane, or clicking the Edit button in the View Object screen.

The user is presented with an Edit Object screen (See figure 14.3.1). The screen is essentially the same as the second stage of the Add Child Object screens, with text boxes filled, of course. All the sections are displayed in an expanding state by default. The user can switch beween the expanding state and callapsing state by clicking the icon next to each section header.

Figure 14.3.1 – Example screen shot for editing a Phenotype object


Parent Details

The Parent Details section is exactly the same as in the Object View Page, with the parent object ID, title, type and relationship displayed in a read only mode.


Object Details

The Object Details section displays all the attributes of the object. Figure 14.3.2 shows an example of a Platform object being edited. Object Title and Decription can be modified. Some other fields may be available for editing depending on the type of the object. At the bottom of the Object Details section, Creator, Creation date, Last modified by and Last modification date are displayed in a read only mode (which is not in the Add a Child Object page).

Figure 14.3.2 – Screen shot of a Platform object details being edited


RefersTo Tables

RefersTo Tables can be edited. All the available objects are listed under each RefersTo table. The originally selected object can be de-selected. New object can be added for referencing.


File Attachment

The attached files are listed as in the View Object page. More files can be attached to the object. The procedure is the same as Attach Files in the Create a Project section.

* Note the user cannot edit attached files or information about the attached files in this screen, they would have to select the individual attached files.


Action Buttons

When satisfied with the changes, the user clicks Submit.

Validation is performed.

If any fields are in error then the user is returned to the top of the page. An error box will appear at top of the page advising there are errors on the page. An error box will appear beneath the fields in error describing the nature of the error and remedial action.

The system confirms the submission to the administrator by switching to the view tab and displaying the object details page in view (read only mode).

If the user clicks the Cancel button, the Edit Object screen will be abandoned.  The screen is returned to the View Object page without any change.

The user can also click the Reset button to reset the field values back to the origianal value and the screen will remain on the Edit Object page.

If there is another object is being edited by the same user, the user has to click the ReInitialize button to edit the object.

Maintaining a Child Object


Overview

A child object means an object of another object. In PODD, every object is an child object of a parent object except Project object.

Child Objects are defined as Concrete Objects. They are an instance of a Concept Object. This section will describe how a concrete object is initiated and maintained as Child Objects of a Parent Object, including

  • Adding a Child Object
  • Viewing a Child Object
  • Editing a Child Object
  • Deleting a Child Object

Actors

For repository users:

  • Public users can view objects descended from public access projects.
  • Registered users can view objects descended from public access projects and may access objects descended from other projects as determined by their project role.
  • Project Creators are as per registered users.
  • Administrators can add/edit/view/delete/undelete objects.

For users with project specific roles:

  • Project Administrators can add/edit/view/delete objects descended from a project object they have admin access to.
  • Project Members may can/edit/view objects descended from a project object they have member access to.
  • Project Observers can view objects descended from a project object that they have observer access to.

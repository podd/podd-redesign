Viewing a Child Object


The View Object screen is arrived as a result of Adding a Child object or Editing an Object. It can also be displayed by selecting the object’s View link from the Browser Pane or click the object title anywhere (object hirarchy, clipbord, refersTo table etc).

The View Object screen is displayed as follows (See figure 14.2.1). The following sections are displayed in the View Object page:

  • Parent Details,
  • Object Details,
  • RefersTo tables (could be zero or more than one depending on the object type),
  • File Attachemnt,
  • Hierarchy.

All the sections are displayed in an expanding state by default. At the bottom of the page, available action buttons are listed for the user.

Figure 14.2.1 – Example screen shot for viewing an Investigation object & relationships.


Parent Details

The first section of the View Object screen is the Parent Details section. It is exactly the same as described in the Adding a Child Object screen, including the parent object's ID, type, Title and relationship with the child object.


Object Details

The Second section is the Object Details section. It presents the available attributes of the object being viewed. The title of this section is determined by the object type. Diferent Object type has different object details. 


Refers To Tables

Refers To table is displayed only when the data model permits. i.e., a Material Object may refer to an Environment object, a Genotype object, a protocol object and a Treatment Object. A Project Plan will not refer to any other objects and this section is not visible. Even no refersTo objects are selected, the empty table is still displayed.


File Attachment

The next section is the File attachments section. All the attached files are listed with their File Name, Size, Type, Format and Description. The File details can be viewed by clicking the file name link. The user can also download an individual file or all files together.


Hierarchy

The Hierarchy section is the similar to the one in the Viewing Project screen except that the project object does not have a parent object. As stated under the Hierarchy header: Hierarchy displays only the parent and existing children of the current object. It displays entries for:

  • The current object's Parent Object;
  • The current object; and
  • The current object's children.

Action Buttons

The bottom of the page displays some action buttons, Edit Object, Add Child Objects, Copy, Paste and Delete.  Whether a button is visible or invisible depends on the object’s project Status, the user’s repository role and project specific role, and the object’s position in the domain model as well.  

  • The Edit button takes the user to the Edit Object page for the current object (it is only displayed in Active project, and only available to repository administrator, project administrator and project member). It will be described in next section: Editing an Object.
  • The Add Child Objects will make this current object the current parent object and will take the user to the Add Child Object screen as described in the previous section: adding a child Object. (This button will only be available if the model permits. i.e. If a Project Plan does not allow for any child objects, then this button would not be available).
  • The Copy button will allow the authorised user to copy the current object to the Clipboard.
  • The Paste button will brings the authorised the user to the Clipboard where the user can select objects/predicates to paste.
  • The Delete button is only available for the repository administrator or project administrator.

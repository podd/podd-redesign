PODD Projects


Overview of Projects

A project is a foundation object in PODD, it sits at the top of the object hierarchy.

Conceptually a project is an abstraction of a real world project, whereby a set of objectives are established, materials are produced, measurements are recorded, observations are made and outcomes are concluded.

A project is not an investigation/experiment, it may contain an investigation, or it may contain thousands of investigations.

Project is a metadata object and not a data object. Data object types may be attached to Project objects.

The Project object will have the following attributes:

  • URI * (System Generated)
  • Title *
  • Description
  • Lead Institution *
  • Start Date *
  • Duration
  • Notes
  • ANZSRC Field of Research Codes *
  • Completion Date (determined by date Project Status is set to complete).
  • Project Status (Active/Inactive/Complete)
  • Publication Status * (Published/Not Published)
  • Publication Date * (determined by date publication status is set to published).
  • Creator *
  • Creation Date *

Project Status

A Project status can be active, inactive or complete. It is marked next to the project title on the View Project Object page. The default status is active.

  • Active means the project is developing. It is not publicly accessible, but the project members have CRU access to the Project and its objects.
  • Inactive means that the project is abandoned. The project and its objects cannot be updated. All project participants will only have read access to the project. Repository wide Administrators will still have CRU access to the project, and thus can revert the status.
  • Complete means the project is finished. Accesses will be the same for the Inactive status. The project and its objects cannot be updated. The project status can be reverted if the publication status is Not Published.

Once a project is complete, the principal investigator or repository administrator can publish the project and make it publicly accessible. Therefore, the publication status of a project can be published or not published.

  • Published means the Project is complete. Public anonymous users and registered users have complete read access to the project and its objects. Once published the project status cannot be changed.
  • Not published means that the project and its objects are only accessible to the people listed as project participants. The project may either be incomplete (active) or complete.

Active and inactive projects can not be published.

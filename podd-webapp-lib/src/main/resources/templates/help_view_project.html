Viewing a Project


Overview

Users can view projects as determined by their repository role or project specific roles (same as Listing Projects): 

  • Public users can view public access projects.
  • Registered users can view public access projects and the projects they are associated with.
  • Project Creators view public access projects and the projects they are associated with. They will not have any view access over the project they created unless they define themselves with a project role during project creation.
  • Administrators can view all projects.

Project View

The authorised user can access the Project View via the following methods:

  • Creating and submitting a new project (the new project object has been instantiated and validated);
  • Clicking the Project Title link on the Listing Projects page;
  • Clicking the View link of the project in the Browser Pane.

Figure 11.1 – Project View (Project Member) in collapsing state

The Project View is displayed in an expanding state by default. Figure 11.1 shows a screen shot of an Registered User's Project View (Project memebr) in a collapsing state to reduce the picture size.

On the View Project page, there are four sections:

  • Project details
  • Project anticipants
  • File attachment
  • Hierarchy

Based to the user’s repository and project specific role, different action buttons will be displayed at the bottom of the page: Edit Object, Paste, Add Child Object, Delete and Publish Project.

  • An Edit Object takes the user to Edit Project Object page. The user can edit the project details, update the project participants, add or delete data items. The same action would be effected by clicking on the edit tab.
  • An Add Child button would take the user to an Add Child Object page, with the project as the parent.
  • A Paste button takes the user to the clipboard screen. The user can select objects/predicates to paste to the Project if there are any previously copied objects/predicates available on the screen.
  • A Delete button allows the authorised user to delete the project if there is no child object or all child projects have been already deleted.
  • A Publish Project button is displayed on the View Project Object page only when the Project status has been set to Complete.

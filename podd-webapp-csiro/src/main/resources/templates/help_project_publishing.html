Publishing a Project


Overview

When a project is complete, the user may choose to publish the project. No information in a project is publicly available until the project is flagged as published. 

This section will show you how to publish a project. The behaviour of the system is also described when a project is published. It includes the checks that are required and the additional functions (e.g. allocating ANDS PIDS) that are to be executed at the time of publication.


Actors

  • Public users cannot modify projects at all.
  • Registered users cannot update a project's status and publish a project, unless they have an appropriate project specific role.
  • Administrators may update a project's status and publish a project.

For users with project specific roles:

  • Principal Investigator may update a project's status and publish a project.
  • Project Administrators may update a project's status and publish a project.
  • Project Members may modify and view a project object, but cannot update a project's status and publish a project.
  • Project Observers cannot update a project's status and publish a project.

How to Publish a Project


Set the Project status to complete

The Publication Status field is not available to update until the Project Status field is set to complete.

Navigate to the Edit Project page, change the project status to Complete and then click the Submit button at the bottom of the page. A new View Project Object page is then returned (Figure 13.1).

Figure 13.1 – Sample Screen of Project View of a complete project

Figure 13.1 shows the Project View of a complete project in a Project Administrator/Principal Investigator's view. It is displayed in a read only mode. A Publish Project button is displayed on the page.

For Project members and Project Observers, the Project is also displayed in a read only mode, the Publish Project button is not displayed.

Only a Repository Administrator can update a Completed Project.


Publish the project

When a project is completed, a Repository Administrator or a Project Administrator/principle Investigator can set the Publication Status field to Published by clicking the Publish Project button at the bottom of the View Project Object page (Figure 13.1).

Once the Publish Project button is clicked, the request is submited to the system. A new screen will be displayed advising that the project has been successfully published (Figure 13.2). 

Figure 13.2 – Screen shot after the project is published

At the same time, an email is sent to the project administrator notifying that the Projet is published. The email reads as follows:

Dear firstname lastname,

The following project has been published in PODD and is now publicly available.

  ID: poddObject: 10034

  Title: Tab Test 1 - Arabidopsis stomatal closure mutants OST1-2 growth analysis

  Description: Comparison between OST1-2 mutants and Ler wild types under various conditions

You should receive an email notifying you of the projects persistant URL shortly.


Project View of a Published Project

Click the Back to Project button on the page (Figure 13.2), an updated View Project object page is displayed.

All fields in all objects can no longer be updated by anyone except the Repository Administrator (The Repository Administrator can also revert the publication status and project status).

  • All objects in the project hierarchy, including the project object, will become publicly available.
  • The Publised project is still listed under My Project header when the project participants list the projects.
  • Paste,  Edit Object and Add Child Object buttons are not available on the View Object page. Therefore, no objects or data files can be pasted to the published project and its child objects. Data files can still be downloaded and child objects can be copied. 
  • In the Browser Pane, all the Edit/Add Child links are not visible. Only the View and Copy links are displayed.
  • All objects will be registered with the ANDS (Australian National Data Service) Persistent Identifier Service.
